# GitHub i18n Next.js 管理平台 PRD

## 1. 项目概述

### 1.1 项目名称
GitHub i18n Admin - GitHub 中文翻译管理平台

### 1.2 项目类型
Next.js Web 应用 - 浏览器用户脚本的配置管理和监控平台

### 1.3 核心价值主张
为 GitHub 中文翻译插件提供现代化的 Web 管理界面，实现配置管理、性能监控、词典维护等功能的可视化操作。

### 1.4 目标用户
- GitHub 中文翻译插件用户
- 插件开发者
- 翻译词库维护者

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 配置管理中心
- 用户配置管理（启用/禁用功能）
- 性能参数调整（缓存大小、批处理大小等）
- 翻译模式配置（部分匹配、完整翻译等）
- 更新检查设置

#### 2.1.2 性能监控仪表盘
- 实时翻译统计
- 缓存命中率图表
- 翻译延迟监控
- DOM 操作统计

#### 2.1.3 词典管理
- 词典导入/导出
- 词典条目查看和编辑
- 常用词典管理
- 页面特定词典（Codespaces, Explore等）

#### 2.1.4 关于页面
- 版本信息
- 更新日志
- 项目链接
- 问题反馈

### 2.2 用户交互流程

```
用户访问首页
    ↓
    ├─→ 配置管理（基本设置/性能设置/更新设置）
    ├─→ 性能监控（统计图表/实时数据）
    ├─→ 词典管理（查看/编辑/导入导出）
    └─→ 关于（版本/文档/反馈）
```

### 2.3 数据存储
- 使用浏览器 localStorage 存储配置
- 支持配置导入/导出为 JSON 文件
- 性能数据本地缓存

## 3. 技术架构

### 3.1 技术栈
- **框架**: Next.js 14+ (App Router)
- **UI**: Tailwind CSS + shadcn/ui
- **图表**: Recharts
- **状态管理**: React Context + localStorage
- **部署**: Vercel / Netlify / Docker

### 3.2 项目结构
```
next-app/
├── app/
│   ├── page.tsx              # 首页
│   ├── layout.tsx            # 布局
│   ├── config/                # 配置管理页面
│   ├── monitor/               # 性能监控页面
│   ├── dictionary/            # 词典管理页面
│   └── about/                 # 关于页面
├── components/
│   ├── ui/                    # UI 组件
│   ├── layout/                # 布局组件
│   ├── config/                # 配置组件
│   ├── monitor/               # 监控组件
│   └── dictionary/             # 词典组件
├── lib/
│   ├── utils.ts               # 工具函数
│   ├── config-manager.ts      # 配置管理
│   └── storage.ts             # 存储管理
├── hooks/
│   ├── useConfig.ts           # 配置 Hook
│   └── useStats.ts            # 统计 Hook
└── types/
    └── index.ts               # 类型定义
```

### 3.3 响应式设计
- 桌面端：完整导航 + 侧边栏
- 平板端：折叠导航
- 移动端：底部导航

## 4. UI/UX 设计方向

### 4.1 视觉风格
- 现代简约风格
- GitHub 设计语言一致性
- 暗色模式支持

### 4.2 配色方案
- 主色：GitHub 蓝色 (#0969da)
- 背景：浅灰/深灰自适应
- 强调色：绿色成功、红色错误、黄色警告

### 4.3 布局
- 左侧导航 + 右侧内容区（桌面）
- 底部导航 + 全屏内容区（移动）
- 卡片式内容组织

## 5. 非目标（Out of Scope）

- 不实现浏览器翻译功能（保留用户脚本）
- 不实现服务器端存储
- 不实现用户认证系统
- 不实现实时协作功能

- ✅ 性能统计图表正确显示
- ✅ 词典导入/导出功能可用
- ✅ 响应式设计适配移动端
- ✅ 构建和部署成功

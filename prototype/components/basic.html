Components · Basic

基础组件

构成界面的最小原子单元——按钮、开关、输入框、徽章与标签。

按钮

PRIMARY / SECONDARY / DANGER / GHOST
主要按钮用于高优先级操作;次要按钮用于中性操作;危险按钮用于不可撤销的操作;幽灵按钮用于弱强调。尺寸通过 btn-sm / btn-lg 控制。

开关

ON / OFF
开关用于即时生效的二元选项,通过 is-on 控制状态。建议左侧放置标签,右侧放置开关。

输入框

DEFAULT / FOCUS / DISABLED
输入框默认使用内凹背景聚焦时显示蓝色光环。通过 disabled 属性或降低透明度来呈现禁用态。

徽章

STATUS / TAG
已翻译 info warn error
徽章用于标记状态或分类信息。语义色映射:success→绿色,info→蓝色,warn→橙色,danger→红色。

使用规则

Design guidelines · 使用基础组件时请遵循以下原则

  • 层级单一:同一操作区域内避免同时出现多个主要按钮,主按钮最多 1 个,其余使用次要或幽灵样式。
  • 状态即反馈:开关、输入框在状态切换时应立即反馈,不出现加载或二次确认,除非是危险操作。
  • 语义色一致:徽章与语义色必须一一对应,绿色=成功/已翻译,蓝色=信息,橙色=警告,红色=错误。
  • 禁用态明示:任何禁用的组件都应降低透明度或提供工具提示,告知用户"为何不可用"。
  • 尺寸统一:相同行高的组件应保持垂直基准线对齐,使用 --sp-* 间距变量而不是像素值。

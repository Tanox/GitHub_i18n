使用规则

使用规则

  • 首屏信息密度控制:默认只展示核心项,不超过 3–5 个,其余通过"展开更多"获取。
  • 展开状态需记忆:用户展开的分组在同一会话内应保持展开,不要每次进入都重新收起。
  • 清晰的折叠标识:使用箭头(▸/▾)或加号(+)明确表达"可展开/可收起",避免仅靠颜色暗示。

撤销操作 · Undo

Toast with action · Non-destructive confirmation

与其让用户在操作前确认,不如让他们能够在操作后撤销——更轻量,也更符合"先行动后思考"的直觉。

✓ 已恢复默认设置 撤销
已将「repository」从词典中移除 撤销
! 已删除自定义词典 user.json 恢复

使用规则

  • Toast 生命周期:撤销入口在 6–10 秒内有效,超时后自动隐藏或显示"已过期"。
  • 不可逆操作不使用撤销:涉及密码重置、权限撤销等安全操作,应使用确认对话框而非撤销。
  • 语言明确:"撤销"一词对应英文 Undo,不与"取消"(Cancel)混用,后者用于中止当前操作。

键盘导航 · Keyboard Shortcuts

Keyboard-first · Accessibility

为高频操作提供键盘快捷键,让高级用户可以摆脱鼠标,同时为辅助技术提供清晰的焦点状态。

快捷键 功能 适用场景
Ctrl + B / ⌘ + B 打开/关闭配置面板 全局 · 任意页面
Ctrl + T / ⌘ + T 立即翻译当前页面 全局 · 任意 GitHub 页面
Esc 关闭当前面板或模态 全局 · 面板打开时
↑ / ↓ / ← / → 在列表或表格中切换焦点 词典管理、词条列表
Enter 激活当前聚焦项或保存编辑 表单、行内编辑
/ 聚焦到全局搜索框 全局 · 搜索可用时

使用规则

  • 与平台约定一致:macOS 使用 ⌘,Windows/Linux 使用 Ctrl,快捷键文档同时列出两者。
  • 可见的焦点环:所有键盘可交互元素在聚焦时都必须有清晰的蓝色光环,不可使用 outline: none 去掉焦点。
  • 避免与浏览器冲突:不要劫持 Ctrl+W、Ctrl+L、F5 等浏览器默认快捷键。

表单验证 · Validation

Inline validation · Real-time feedback

表单验证在输入过程中即时反馈,告知用户"什么是对的、什么还需要改",而不是等到提交时一次性报错。

词典名称
✓ 格式正确,该名称可用
词典路径
! 路径以斜杠结尾,建议改为相对路径或包含文件名
翻译语言
✗ 请使用 ISO 639-1 代码,例如 zh-CN

使用规则

  • 失焦后首次校验:输入框在用户第一次 blur 后开始校验,而非每次按键都报红,避免打扰输入流。
  • 三色语义明确:绿色=通过,橙色=可改进但可提交,红色=不可提交,仅红/绿时使用会失去中间态。
  • 错误消息要可行动:不说"输入错误",而说"请使用 ISO 639-1 代码,例如 zh-CN",告诉用户"怎么改"。

MOTION · 动效

动效与动画规范

克制而有意义的动效用于强化感知与反馈,不炫技,不喧宾夺主。 动画应当传达空间层级,而不是装饰本身。

动画时长规范

Duration · 毫秒级的节奏控制

名称 时长 使用场景
Fast 120ms 按钮/开关反馈。最接近瞬时的响应,给用户"立即响应"的感觉,不可感知到延迟。
Base 200ms 卡片悬停、面板展开。常见的微交互时长,可感受到变化但不打断流程。
Slow 320ms 页面过渡、模态入场。用户注意力聚焦的场景,动画需要足够明显以表达层级。
Long 500ms 数据加载、图表动画。需配合进度指示,不应用于按钮等需要即时响应的场景。

缓动函数

Easing · 动画的节奏与个性

Ease-out · 快出慢停

cubic-bezier(.22,1,.36,1)
用于进入界面、响应用户点击的元素。开始快速、末端缓停,给人自然的"到位"感。

Ease-in-out · 平滑进出

cubic-bezier(.65,0,.35,1)
用于状态切换、页面内元素的位移。两端对称的加速曲线,适合循环或双向过渡。

Spring · 弹性回弹

cubic-bezier(.34,1.56,.64,1)
用于按钮缩放、标签浮起。轻微的过冲带来活泼与愉悦,但需克制使用,避免轻浮。 
/* CSS 变量定义 */
--ease-out: cubic-bezier(0.22, 1, 0.36, 1);       /* 进入 / 点击响应 */
--ease-in-out: cubic-bezier(0.65, 0, 0.35, 1);    /* 状态切换 / 双向 */
--ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1); /* 弹性交互 / 按钮 */

transition: transform var(--t-base) var(--ease-out);

每个动效对应一个 CSS 类名,可直接添加到任意元素上。组合使用时建议遵循"一次只跑一个主要动画"的原则,避免视觉噪音。 
/* 淡入 - 用于内容出现 */
.motion-fade { animation: fadeIn var(--t-slow) var(--ease-out) both; }
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(8px); }
  to   { opacity: 1; transform: translateY(0); }
}

/* 滑入 - 用于侧边面板 */
.motion-slide { animation: slideIn var(--t-slow) var(--ease-spring) both; }
@keyframes slideIn {
  from { transform: translateX(-30px); opacity: 0; }
  to   { transform: translateX(0); opacity: 1; }
}

/* 悬停缩放 - 用于卡片 */
.motion-scale { transition: transform var(--t-base) var(--ease-spring); }
.motion-scale:hover { transform: scale(1.08) rotate(-2deg); }

/* 脉冲 - 用于持续提示 */
@keyframes pulse {
  0%, 100% { box-shadow: 0 0 0 0 rgba(46,160,67,.25); }
  50%      { box-shadow: 0 0 0 14px rgba(46,160,67,0); }
}

使用规则

Motion guidelines · 让动效有意义

  • 同一时间只允许 1–2 个动画同时运行,避免视觉噪音。当多个元素都在动时,用户的注意力会被分散,页面显得"忙乱"。优先让最重要的元素动起来。
  • 页面首屏加载避免超过 1 秒的动画,确保内容快速可达。首次渲染的重点是内容呈现,动效应作为点缀而非主角。超过 500ms 的入场动画需重新审视。
  • 可访问性:尊重 prefers-reduced-motion,禁用装饰性动画。对于有前庭障碍的用户,动画可能造成不适。所有装饰性动画必须在检测到 reduced-motion 偏好时降级为淡入或直接跳过。
  • 反馈类动画(如按钮点击)时长控制在 120ms,避免打断用户。反馈的核心是"告知",不是"表演"。用户按下按钮后希望立即看到结果,不应等待动画完成。
  • 动画应表达空间层级:父级缓动长于子级,形成层次递进。容器的过渡速度慢于内部元素,子级在父级之后以稍快的速度跟进,模拟物理世界的惯性,强化"谁包含谁"的层级关系。

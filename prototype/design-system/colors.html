使用规则

Usage rules · 何时使用哪一种颜色

  • 主色 green 仅用于主操作按钮与品牌强调。为保证焦点明确,同一页面内主色按钮不超过 2 个。
  • 强调色 blue 用于链接、激活态与信息类徽章。在 GitHub 原生语境中,蓝色代表"可点击"。
  • 危险色 red 保留给破坏性操作与错误状态。删除、覆盖、无法撤销的操作才使用红色。
  • 中性色遵循层次:背景 → 前景背景 → 表面 → 边框 → 文本。越往上越亮,形成清晰的纸面层次。
  • 文本需满足 WCAG AA 对比度。主文本与背景对比度不低于 4.5:1,次文本不低于 3:1。
  • 语义色作为背景时,必须搭配对应的前景色使用。例如 success-bg 搭配 success-fg,避免在浅色背景上再叠浅色文字。
  • 紫色与粉色用于装饰性区分而非功能性表达。如分类标签、特殊强调,不在交互态中使用。
